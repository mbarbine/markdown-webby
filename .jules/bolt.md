## 2024-05-24 - Zustand Un-selected Store Re-rendering Anti-pattern
**Learning:** In the `EditorPage` (`app/editor/page.tsx`), the `useMarkdown` store was being called without a selector or `useShallow`, which causes the component to re-render whenever *any* part of the global state changes (e.g., search queries, tree positions). This is a critical performance bottleneck given the frequent global state changes in this specific app (like hovering over nodes in the graph).
**Action:** Always use targeted selectors or `useShallow` from `zustand/react/shallow` when extracting state from the `useMarkdown` store (and other Zustand stores) to prevent widespread unnecessary re-renders.
