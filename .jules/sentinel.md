## YYYY-MM-DD - JSON-LD XSS Prevention
**Vulnerability:** XSS vulnerability in `JsonLd` component. When embedding JSON-LD schemas, `JSON.stringify` does not escape `<` and `>` characters by default. If untrusted data is included, an attacker could break out of the script tag (e.g., `</script><script>alert(1)</script>`).
**Learning:** `dangerouslySetInnerHTML` on script tags with `JSON.stringify` requires escaping HTML characters, particularly `<`.
**Prevention:** Replace `<` with `\u003c` or use a dedicated serialization library like `serialize-javascript` when interpolating JSON directly inside a script tag.

## 2025-02-23 - Markdown Rendering XSS Prevention via Attribute Breakout
**Vulnerability:** XSS vulnerability in `components/editor/markdown-preview.tsx`. When rendering user-provided Markdown into HTML, the custom `renderMarkdown` function failed to sanitize image URLs (`src`) and link URLs (`href`). Additionally, quotes in user-controlled variables were not escaped, allowing attribute breakout (e.g., `![alt](" onerror="alert(1))`). Dangerous URL protocols (`javascript:`) were also unhandled.
**Learning:** Preventing XSS in custom Markdown renderers requires multiple layers of defense: 1) HTML entity encoding for textual content, 2) Escaping quotes (`"` and `'`) for attribute values to prevent breakout, and 3) Stripping dangerous URL protocols (`javascript:`, `vbscript:`, `data:`) from attributes like `href` and `src`. Note that `data:image/` URIs are generally safe and often required in Markdown rendering, so protocol blocking must be context-aware.
**Prevention:** Always use a helper function (like `sanitizeAttribute`) to ensure quotes are safely encoded. Before checking for dangerous protocols, normalize the URL by removing HTML entities and control characters to prevent evasion tactics (e.g., `java&#x09;script:`).
